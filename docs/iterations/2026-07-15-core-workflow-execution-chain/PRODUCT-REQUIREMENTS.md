# 核心迭代工作流执行链产品需求

## 背景

AgentForge 已具备 `Project -> Session -> PipelineRun -> StageState -> Artifact -> Delivery -> Eval Feedback` 状态主链路，但阶段语义、任务执行、真实代码修改、测试门禁和阶段推进仍未形成一个可验证的执行闭环。当前系统可以展示“处于哪个阶段”，却不能稳定证明“该阶段消费了哪些上游结论、执行了哪些任务、修改了哪些文件、测试是否真的通过”。

## 产品目标

面向全栈开发工程师，把一次需求从分析到交付串成可追溯、可暂停、可恢复、可验证的执行链：

1. 每个阶段有明确输入、目标、产物和完成标准。
2. 任务拆解产生结构化 TaskGraph，而不是仅生成 Markdown。
3. 开发阶段只能在用户授权的 ProjectMount 中生成并应用文件级变更。
4. 测试阶段执行真实命令，失败会阻止交付完成。
5. 确认、阶段推进、Run 完成和新需求创建由统一 Orchestrator 管理。
6. 全链路具备用户隔离、审计、回放和浏览器验收。

## 非目标

- 不一次性重写现有 Pipeline、Skill Engine 或 Delivery 子系统。
- 不把工作流固定成只适用于 AgentForge 仓库的脚本。
- 不允许平台自动扫描未授权目录。
- 不以聊天消息文本替代结构化执行事实。

## 验收结果

完成路线图后，一次迭代必须能回答：需求为何被分类到该 Pipeline、每个 Stage 消费了什么、TaskGraph 如何生成、文件如何修改、测试命令和结果是什么、谁确认了哪些风险、最终 Artifact 如何交付，以及失败后从哪里恢复。
