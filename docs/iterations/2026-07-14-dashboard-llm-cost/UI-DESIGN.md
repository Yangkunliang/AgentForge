# TASK-046 UI 设计：Dashboard LLM 用量

## 设计方向

Dashboard 面向需要持续迭代项目的全栈开发工程师，采用安静、紧凑、可扫描的运行观测面板，不做营销式大标题、装饰插画或夸张卡片。

## 信息层级

在任务费用与 Agent / Skill 区域下方增加一张“LLM 实际用量”面板：

1. 第一行展示四项等权指标：已记录调用、累计成本、Token、平均延迟。
2. 第二行分为“按 ModelRoute”和“按 Stage”两列排行。
3. 每行排行展示名称、调用次数、Token 和成本，成本使用等宽数字并右对齐。

旧“今日费用”标题改为“任务费用（今日）”，使两种费用口径在首屏即可辨认。

## 状态

- 加载中：沿用页面现有 loading 遮罩，不引入布局跳动。
- 零数据：四项指标显示 0，两个排行区域显示“暂无 LLM 调用记录”。
- 有数据：最多显示前 3 项，长名称截断并通过 `title` 保留完整值。
- 数值：成本保留 4 位小数；Token 使用千分位；延迟低于 1000ms 显示毫秒，否则显示秒。

## 响应式与可访问性

- 桌面端四列指标、两列排行。
- 窄屏下指标变为两列，排行上下堆叠。
- 数字区域使用稳定网格和 tabular numerals，动态内容不会改变整体宽度。
- 排行使用语义化 section 和列表结构，颜色只做辅助，不承担唯一信息表达。
