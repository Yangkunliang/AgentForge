# TASK-042 UI 设计：Dashboard 授权指标

## 设计原则

Dashboard 是工作型页面，本次保持安静、密集、可扫描的运营面板风格，不做营销式大卡片或强装饰。

## 页面结构

在现有第二行统计区下方新增一张「高风险 Skill 授权」卡片：

- 顶部：标题 + 总览数字，展示授权请求、已授权、通过率。
- 中部：一条细进度条表达授权通过率。
- 底部：两列排行，左侧按 Skill，右侧按 permission。

## 状态设计

- 无事件：数值为 0，排行区域显示「暂无授权记录」。
- 有事件：排行显示名称、required/granted 和通过率。
- 高通过率不直接标绿为“好”，低通过率也不直接标红为“坏”，避免把策略严格度误解为异常。

## 响应式

桌面端两列排行；窄屏下上下堆叠。所有数字和长名称必须换行或截断，不影响页面布局。
