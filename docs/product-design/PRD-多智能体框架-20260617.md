# 多智能体协同工作框架 - 产品需求文档 (PRD-v1)

## 1. 产品定位

**一句话定义**：面向全栈开发者的多智能体协同工作框架，让 AI Agent 像人类团队一样协作完成任务。

**核心理念**：借鉴 **Harness Engineering**（模型外环境）理念，公式：**Agent = Model + Harness**
- Model 提供"思考"能力
- Harness 负责"让它真的能干活的工程支撑"

**目标用户**：个人开发者（全栈工程师）→ 后续可扩展至团队协作场景

## 2. 用户故事

| ID | 作为... | 我想要... | 以便于... |
|----|---------|----------|----------|
| US-1 | 全栈开发者 | 输入一个自然语言需求 | 系统自动拆解任务并分派给合适的 Agent |
| US-2 | 全栈开发者 | 让 Agent 自动完成产品设计、UI 设计、任务拆解 | 减少重复性工作 |
| US-3 | 全栈开发者 | 快速导入新的 Skill 插件 | 扩展 Agent 能力 |
| US-4 | 全栈开发者 | 导出训练数据 | 优化我的模型和 Agent 路由 |
| US-5 | 全栈开发者 | 看到安全的 API 调用记录 | 审计和合规 |

## 3. 核心功能

### 3.1 多 Agent 协同
- **动态协商**：Agent 间通过 Contract Net 协议自动分工
- **能力路由**：基于 Skill 注册中心自动匹配最佳 Agent
- **降级策略**：主 Agent 失败自动切换备选

### 3.2 Skill 插件系统
- **本地热加载**：`skills/` 目录变更自动刷新
- **远程安装**：`pip install` 从 PyPI/仓库安装
- **双模式**：YAML 声明 + Python 实现

### 3.3 安全管控
- API 认证（JWT + API Key）
- 限流（Token Bucket）
- Prompt 注入防护
- 全链路审计日志

### 3.4 数据导出
- 训练数据收集（JSONL 格式）
- 自动脱敏
- 模型优化指标分析

## 4. 非功能需求

| 类别 | 要求 |
|------|------|
| 性能 | API 响应时间 < 5s（简单任务），< 30s（复杂任务） |
| 安全 | JWT 认证 + API Key 轮换 + 限流 |
| 可扩展 | Skill 插件热加载，Agent 动态注册 |
| 可观测 | 全链路 trace_id + 审计日志 |

## 5. 技术栈决策

| 组件 | 决策 |
|------|------|
| 后端 | Python + FastAPI (async) |
| LLM 网关 | LiteLLM（统一 API + cost 追踪） |
| 消息队列 | RabbitMQ（持久化 + ACK + 死信队列） |
| 数据库 | PostgreSQL（开发/生产统一） |
| 认证 | JWT + API Key |
