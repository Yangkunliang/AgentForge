## 0. 背景与愿景

### 0.1 命名来源

**CLAW**（**C**onnected **L**ayer for **A**gent **W**orkflows）是 AgentForge 的外部集成能力层。  
就像螃蟹的钳子，CLAW 让 Agent 能真正"抓取"外部世界的信息和服务，而不是凭空捏造。

当前 AgentForge 已具备：
- ✅ 多 Agent 协同框架（Contract Net + Harness 六层）
- ✅ 基础 Skill 引擎（ReAct 循环，天气 Skill 内置）
- ✅ 流式聊天 UI（SSE + Vue 3）

CLAW 要解决的核心问题：**Agent 的"爪子"太短**，只能用训练数据，无法触达开发者每天真正使用的工具生态。

### 0.2 目标用户价值

面向全栈开发者，AgentForge 的 Agent 应该能：
- 查 GitHub PR、Issues、代码
- 查数据库、读写本地文件
- 在 Jira/Linear 创建任务
- 读 Slack/Email 上下文
- 执行代码并返回结果
- 搜索最新文档和 Stack Overflow

这些能力通过 **MCP（Model Context Protocol）** + **第三方 Skill** 两条路径交付。

## 4. L2 MCP 官方 Server 集成

MCP（Model Context Protocol，Anthropic 开源协议）提供了标准化的工具调用接口，大量优质 Server 已由官方和社区维护。

### 4.1 集成架构

AgentForge 后端作为 **MCP Client**，通过以下方式连接 MCP Server：
- **stdio 模式**：本地进程（`npx`/`uvx` 启动）
- **SSE 模式**：远程 HTTP Server（云端 MCP）

```python
# 配置示例（mcpServers 字段写入 agent_config 或全局 settings）
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": { "GITHUB_PERSONAL_ACCESS_TOKEN": "..." }
    },
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/workspace"]
    }
  }
}
```

### 4.2 推荐 MCP Server 列表（全栈开发者场景）

### 4.3 MCP 集成实现方案

AgentForge 后端使用 `mcp` Python SDK 实现客户端：

```
pip install mcp
```

**核心流程**：
1. Agent 启动时，读取其 `mcp_servers` 配置
2. 通过 `ClientSession` 连接对应 Server，获取工具列表
3. 将 MCP 工具动态转换为 LiteLLM tool 格式注入 LLM 请求
4. LLM 返回 tool_call 时，通过 MCP Client 路由到对应 Server 执行
5. 结果作为 tool_result 追加到对话，继续 ReAct 循环

## 5. L3 社区 Skill 市场（ClaWHub）

### 5.1 愿景

**ClaWHub**（`clawhub.ai`）是面向 AgentForge 的社区 Skill 分发平台，类比 npm 之于 Node.js。

> 注：`clawhub.ai` 已在 PRD Skill Engine 中作为市场来源引用，本文档将其完整规划。

### 5.2 功能范围

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 浏览/搜索 | 按分类、标签、Stars 浏览 Skill | P1 |
| 一键安装 | 从 GitHub URL 或包名安装 | P0（已规划）|
| Skill 规范 | `skill.md` + `skill.py` 双文件规范 | P0（已有）|

仅接入**读取**：从 `clawhub.ai` API 拉取 Skill 列表用于展示，安装通过 GitHub URL 完成。不自建市场后端。

## 6. 技术实现优先级

基于用户价值和实现复杂度的综合排序：

### 阶段一：工具基础（2 周）

1. **`web_search` Skill**（Tavily，免费 Key）— 最高频需求
2. **`http_request` Skill**（无依赖）— 通用 REST 调用
3. **MCP 客户端基础设施**（`mcp` SDK 集成到 Harness Executor）
4. **`@mcp/server-filesystem` 接入**（本地文件读写）
5. **`@mcp/server-fetch` 接入**（URL 抓取，无 Key）

### 阶段二：开发者工具（2 周）

6. **`@mcp/server-github` 接入**（代码搜索、Issue 管理）
7. **`@mcp/server-postgres` 接入**（数据库查询）
8. **`code_executor` Skill**（沙箱执行代码）
9. **MCP Server 配置 UI**（Admin 面板，填写 env/command）

### 阶段三：协作工具（持续迭代）

10. **Slack / Linear MCP**（团队协作）
11. **Notion / PDF 读取**（知识管理）
12. **ClaWHub 市场浏览 UI**（Skill 发现）

## 7. 安全与权限设计

CLAW 能力越强，安全边界越重要。

| 风险 | 缓解措施 |
|------|----------|
| 文件系统越权 | 文件 Skill 只允许访问白名单目录（`ALLOWED_PATHS` env）|
| SQL 注入/数据泄露 | DB Skill 仅允许 SELECT，禁止 DDL/DML |
| 代码执行逃逸 | `code_executor` 在 Docker 沙箱执行，网络隔离，超时 30s |
| MCP Server 凭证泄露 | 凭证存 `.env`，不进数据库，不出现在 SSE 事件中 |
| Tool Call 数量爆炸 | ReAct 循环上限 `SKILL_MAX_ROUNDS=5`，总时长 120s |
| Prompt 注入通过工具结果 | 工具返回内容做 HTML 转义，不直接拼接到 system prompt |

## 8. Skill 展示 UI（前端）

多 Agent 复杂度对用户透明，但 Skill 调用过程**可选展示**：

```
用户消息: "帮我查一下 AgentForge 仓库最新的 PR"

─────────────────────────────────────────
  🔧 调用工具: github / list_pull_requests
     仓库: yangkl/AgentForge
  ✅ 工具返回: 3 个 PR
─────────────────────────────────────────

根据 GitHub 返回，最新的 3 个 PR 是：
...（正文）
```

SSE 事件规范（新增）：
```json
{ "type": "skill_called",  "skill": "github", "tool": "list_pull_requests", "args": { "repo": "..." } }
{ "type": "skill_result",  "skill": "github", "tool": "list_pull_requests", "summary": "3 results" }
```

前端默认**折叠**显示，点击展开查看详情。不影响核心对话体验。
